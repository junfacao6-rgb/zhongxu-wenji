# 观复书阁视觉系统：年轻化书单 App

本阶段目标已经从“传统文化首页”调整为“供粉丝阅读馆主上传书籍的平台”。视觉必须先服务阅读、选书、续读与后续上传承接，而不是做一个泛传统文化展示页。首页第一屏应当像一个安静、有秩序、可以立刻开始读书的数字书库，打开后能直接找书、续读和进入馆藏。

公开站名锁定为 **观复书阁**，取自《道德经》“万物并作，吾以观复”。公开页面不再使用“中虚”作为站名或品牌字眼；遗留代码名、存储 key 和资源文件名可为兼容暂时保留。

## 参考来源

从 `D:\我的项目\OpenMontage` 中只保留适合阅读平台的规则：

- `OpenMontage/styles/clean-professional.yaml`：清晰、可信、留白充足。
- `OpenMontage/styles/minimalist-diagram.yaml`：浅底细线、单重点色、信息克制。
- `OpenMontage/skills/creative/typography.md`：中文标题和正文层级必须明确。
- `OpenMontage/skills/creative/cinematic.md`：用空间、光影、景深制造高级感。
- `OpenMontage/skills/creative/ink-theater.md`：黑墨、纸色、少量朱砂和金色强调。

不采用 `flat-motion-graphics.yaml` 的紫色、霓虹、快节奏、互联网宣传感。

## 产品定位

观复书阁第一视觉必须让用户理解三件事：

1. 这是馆主上传书籍后给粉丝阅读的平台。
2. 用户进入后主要动作是找书、试读、继续读。
3. 书籍会按八字、六爻、奇门、方法工具整理成长期馆藏。

首页不是营销 SaaS，也不是玄学海报。它应像一间安静、克制、有秩序的数字书斋。

## 当前选定方向

方向名：**高级低饱和书单 App**

核心判断：用户先后否定了暖纸风格、厚重的 GitHub reader-app 风格，以及过于鲜艳的年轻化版本。现在要更像年轻人愿意停留、但有高级感的内容产品：轻底、短路径、低饱和书单封面、紧凑 Bento 信息流、清晰搜索和移动端优先。保留年轻感，但不要糖果色、活动页、促销页气质。

视觉关键词：

- 轻 App 结构
- 顶部搜索
- 上传入口占位
- 低饱和书单封面
- 继续阅读
- Bento 信息流
- 清爽浅底
- 陶土行动按钮
- 鼠尾草绿点缀
- 靛灰封面
- 无衬线 UI
- 分门别类的馆藏

## 色彩

主视觉使用清爽浅底与高对比内容卡片。深色只作为文字、主按钮和封面压暗层，不再出现大块深色横幅或厚重侧栏。原则是：界面先轻，颜色要克制，书单封面给记忆点，按钮给行动。

- 页面底：`#f7f7f2`
- 面板白：`#fffdf7`
- 正文墨：`#191b17`
- 辅助灰：`#697067`
- 石墨：`#20251f`
- 鼠尾草：`#7f927b`
- 陶土：`#8b564a`
- 靛灰：`#475262`

使用规则：

- 石墨用于主按钮、当前选中和少量高优先级状态。
- 陶土用于状态、标签和次级行动。
- 鼠尾草用于搜索焦点、轻背景和少量进度。
- 书籍封面使用陶土、鼠尾草、靛灰、暖褐、深墨等低饱和组合区分体系。
- 禁止青柠、亮橙、电蓝、糖果粉等高饱和颜色做大面积主视觉。
- 禁止大面积紫色、霓虹、星座感渐变、夸张符咒背景。

## 页面结构

首页：

- 首屏必须像一个可用的书库入口，不像介绍页。
- 必须出现馆藏分区、搜索入口、今日续读、当前书目、上传承接。
- 必须提供明确的新读者入口，例如“先从这三本开始”。
- 左侧为馆藏索引，中间为读书控制台，右侧为馆藏状态与上传计划。
- 文案应服务“粉丝怎么开始读书”，减少工程说明、品牌空话和过度玄幻表述。
- 当前背景资产：`public/images/dao-library-hero.png`。

书架页：

- 标题为“上传给粉丝读的书库”。
- 左侧为馆藏索引。
- 右侧按八字、六爻、奇门、方法工具分组。
- 书籍卡片必须像书籍入口，包含封面、分类、状态、页数和进入阅读。
- 书架页应与首页同处一套“轻 App + 彩色书单封面 + 搜索筛选”的系统，不要变成普通浅色列表，也不要回到全页古典纸感或大块深色资料库。

## 排版

- UI 使用系统无衬线，书名和封面可保留少量宋体/serif。
- 不使用负字距。
- 标题不能为了气氛牺牲可读性。
- 手机端主标题必须固定断行，不能横向溢出。

## 后续功能承接

视觉确认后再做：

- 上传书籍
- PDF/图片/Markdown 阅读器
- 阅读记录
- 收藏
- 粉丝权限
- 后台管理

当前阶段不要提前实现登录、支付、论坛和复杂后台。

## 验收标准

- 第一眼能看出这是“书籍阅读平台”。
- 首页与书架页变化明显，不是旧页面微调，也不是传统文化落地页。
- 首页第一屏必须可执行：找书、搜索、继续阅读、进入书架。
- 页面有高级感但不炫技。
- 手机端无文字溢出、卡片挤压、导航错位。
- `npm run lint` 与 `npm run build` 必须通过。

## 2026-07-04 排版与秩序修正

用户明确反馈“字体和排版太丑，字体不要这么大，整齐简洁又高级，同时带一点玄幻感”。当前应以阅读平台的秩序感优先，而不是用大字和亮色制造气氛。

- 字体：UI 使用更克制的中文无衬线栈，站名、主标题、书封面标题少量使用宋/楷/serif 气质字体。
- 字号：主标题固定层级，不再用 `clamp()` 做巨型响应式标题；桌面首页主标题约 2.55rem，书库标题约 2.65rem，详情标题约 2.75rem，移动端统一约 2rem。
- 排版：三栏结构、搜索区、入门三本、继续阅读和书卡栅格都要收紧，减少空洞大块，卡片半径控制在 8px 或以内。
- 氛围：玄幻感来自低饱和墨色、鼠尾草绿、陶土、靛灰和极淡细线网格，不使用高饱和霓虹、夸张星座背景、装饰性符号或大面积渐变。
- 可读性：所有文字字距保持 0，正文行高舒展但字号克制，按钮和卡片文本必须在桌面与移动端都不溢出。

## 2026-07-05 定位重置：问古书斋

平台名称改为 **问古书斋**，副标题为 **一处安静读古籍的地方**。当前阶段只做一个功能：古籍阅读。

前台必须隐藏或移除上传、会员、购买、咨询、学习路径、后台管理、数据看板、工具箱、文章发布、累计统计、最近内容和后续上传承接。用户进入后只需要按分类找到古籍、查看详情、进入阅读。

视觉关键词：安静、克制、米白纸感、书卷气、目录感、手机端优先、只读书、不复杂。主色为米白、宣纸白、浅墨灰、深棕、墨黑。背景使用柔和纸感或纯净米白，不使用强网格、科技感、玄幻感、游戏感、大红大金或大面积高饱和色。

页面结构：

- 首页：问古书斋、`一处安静读古籍的地方`、主文案、`进入书斋`、`查看全部古籍`、古籍分类、推荐阅读、全部藏书入口。
- 藏书页：分类筛选、搜索框、目录式书籍列表、阅读记录。
- 详情页：书名、分类、页数、简介、章节目录、`开始阅读`。
- 阅读页：顶部书名与返回、米白纸张正文、章节目录展开、上一章/目录/下一章、阅读进度本机记录、沉稳暗色模式。
