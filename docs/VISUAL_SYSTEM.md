# 《问古书斋视觉系统》

## 一、总风格

问古书斋的视觉总风格为：

> 纸墨书院 · 数理星盘 · 现代雅器

当前视觉参考锁定为：

> 横卷古画 · 纸本书案 · 精致藏书阁

首页和公共页面应优先靠近“古画横幅 + 米白纸本 + 细线卡片 + 古籍封面 + 朱砂小印”的气质。整体像一个可长期使用的数字书院前厅，而不是博物馆活动页、玄幻工具站或营销落地页。

它不是玄学噱头站，不是普通资料库，也不是黑金预测软件。它应当呈现为一座有秩序的数字书院：前厅读古籍，内室学术数，静室看盘局，案头写笔记，后台管资料。

视觉气质由三层组成：

1. 书斋感：米白、宣纸、宋体、红印、古籍、留白。
2. 术数感：九宫、洛书、八卦、天干地支、暗金线条。
3. 产品感：清晰导航、舒服阅读、规则化卡片、移动端友好。
4. 横卷感：首页首屏使用宽幅古画作为第一视觉，内容区落在图下，不用大块黑色遮罩，不把标题压在图片上。

所有视觉表达都要服务于可信资料、长期阅读、系统学习、案例练习、笔记复盘和报告交付。

## 二、三套视觉空间

### 1. 书斋浅色系统

用于：首页、藏书、阅读器、术语、课程、八字、六爻、梅花、道家。

关键词：

- 米白纸色；
- 淡墨正文；
- 朱砂印记；
- 古铜细线；
- 竹青点缀；
- 书签、目录、纸笺、古籍封面。

书斋浅色系统是平台主视觉。它要安静、克制、适合长期阅读，不能做成营销页或后台仪表盘。

### 2. 术数深色系统

用于：奇门工具、排盘、择时、报告。

关键词：

- 深墨背景；
- 暗金九宫线；
- 玉青辅助信息；
- 朱砂风险提示；
- 星盘、宫位、门星神干。

术数深色系统只用于工具场景，让用户感觉进入“内室观盘”。它不能扩散到全站，避免廉价黑金和压抑感。

### 3. 后台中性系统

用于：admin 管理后台。

关键词：

- 灰白背景；
- 白色面板；
- 清楚表格；
- 明确状态；
- 低装饰、高效率。

后台可以保留品牌色作为轻量点缀，但优先保证资料上传、审核、草稿整理、用户管理等任务的可读性和效率。

```css
--paper-50: #FBF7EE;
--paper-100: #F5ECDC;
--paper-200: #EADCC3;
--paper-300: #D8C3A0;

--ink-900: #241C15;
--ink-800: #3A2E23;
--ink-700: #514234;
--ink-500: #7A6B5A;

--cinnabar-700: #9E3428;
--cinnabar-600: #B94738;
--cinnabar-100: #F3DDD6;

--bronze-700: #7C5835;
--bronze-600: #9A7048;
--bronze-300: #C7A06F;

--jade-700: #3F6757;
--jade-500: #638879;
--jade-100: #DDE9E2;
```

使用规则：

- `paper-50` 用作页面主背景；
- `paper-100` 用作纸面卡片背景；
- `paper-200` 用作分割线、浅色面板、筛选区；
- `ink-900` 用作大标题；
- `ink-700` 用作正文；
- `ink-500` 用作辅助说明；
- `cinnabar` 用于印章、当前状态、重点标签；
- `bronze` 用于按钮、边框、装饰线；
- `jade` 用于学习类标签、道家/梅花类页面点缀。

### 2. 奇门深色

```css
--qimen-bg: #10110E;
--qimen-panel: #171813;
--qimen-card: #202016;
--qimen-line: #7E6138;
--qimen-gold: #D0A868;

--qimen-bluegray: #60717A;
--qimen-text: #F4EAD8;
--qimen-muted: #B9AA91;
```

使用规则：

- `qimen-bg` 用作奇门工具页背景；
- `qimen-panel` 用作工具面板；
- `qimen-card` 用作宫位卡片和结果卡；
- `qimen-line` 用作九宫线、边框、分隔；
- `qimen-gold` 用作天盘、标题、重点；
- `qimen-red` 用作风险、慎用、朱砂提示；
- `qimen-green` 用作可用、较宜；
- `qimen-bluegray` 用作地盘、辅助信息；
- `qimen-text` 用作正文；
- `qimen-muted` 用作说明文字。

奇门页面不使用大面积鲜红、亮绿，不写“吉凶定论”，统一使用“较宜、可用、慎用、不建议、结构显示、倾向、参考、建议”。

- `admin-bg` 用作后台页面背景；
- `admin-panel` 用作后台卡片、表格、表单；
- `admin-line` 用作表格线和分割线；
- `admin-text` 用作主要文字；
- `admin-muted` 用作次要说明；
- `admin-blue` 用作处理中；
- `admin-orange` 用作待处理；
- `admin-red` 用作风险；
- `admin-green` 用作已发布、已通过。

## 四、字体系统

### 1. 品牌标题

品牌标题使用宋体/衬线字体栈：

```css
font-family: "Songti SC", "STSong", "Noto Serif SC", "Source Han Serif SC", serif;
```

适用：

- 问古书斋；
- 奇门问时；
- 八字命理；
- 六爻纳甲；
- 章节标题；
- 书籍标题。

### 2. 正文

正文使用现代无衬线字体栈：

```css
font-family: "PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif;
```

适用：

- 卡片说明；
- 表单；
- 按钮；
- 后台表格；
- 导航；
- 说明文案。

### 3. 古籍原文

古籍原文使用宋体类字体栈：

```css
font-family: "Songti SC", "STSong", "SimSun", serif;
line-height: 2;
letter-spacing: 0.02em;
font-size: 18px;
```

适用：

- 阅读器原文；
- 古籍短摘；
- 章句引用；
- 经典原文。

### 4. 盘面信息

天干地支、八门九星、八神、宫位等盘面信息使用宋体加粗：

```css
font-family: "Songti SC", "STSong", serif;
font-weight: 600;
letter-spacing: 0.04em;
```

适用：

- 九宫盘；
- 宫位卡；
- 干支四柱；
- 时辰列表；
- 盘理依据。

## 五、组件规范

### PageShell

统一页面背景、最大宽度和垂直节奏。

支持视觉空间：

- `default`：书斋浅色；
- `qimen`：术数深色；
- `admin`：后台中性。

所有新页面默认使用 PageShell，避免页面自行定义大面积背景和容器宽度。

### PageHero

用于首页、藏书、学科、课程、奇门工具、我的书斋等页面首屏。

建议 props：

- `eyebrow`：小标签；
- `title`：主标题；
- `description`：说明文案；
- `actions`：按钮组；
- `visual`：右侧视觉；
- `variant`：`paper`、`qimen`、`minimal`。

书斋页 Hero 使用纸纹、淡金线、朱砂印；奇门页 Hero 使用深墨、暗金九宫线；后台页尽量简洁。

### SectionTitle

用于区块标题。

结构：

- 小标签；
- 主标题；
- 副标题；
- 可选右侧操作。

样式：

- 左侧可有小红印或短线；
- 主标题使用品牌标题字体；
- 副标题使用正文无衬线；
- 行距克制，避免营销页式夸张排版。

### InkCard

用于书籍、课程、术语、学习路径、笔记、报告等常规卡片。

样式：

- 纸色背景；
- 1px 淡金边框；
- 轻微阴影；
- hover 时边框变古铜；
- 圆角保持克制，建议 8px 到 12px。

### SealBadge

用于版权、会员、草稿、审核、私密等状态标签。

常用 tone：

- `cinnabar`：重点、当前、待审核；
- `bronze`：会员、课程、推荐；
- `jade`：已读、可用、已发布；
- `muted`：私密、隐藏、草稿；
- `danger`：风险、退回；
- `success`：通过、完成。

标签不要过大，不要像电商促销贴纸。

### SubjectGateCard

用于学科入口。

每张卡显示：

- 学科名；
- 一句话定位；
- 学习路径前三步；
- 相关书籍数；
- 相关课程数或术语数；
- 进入学习按钮。

学科色彩建议：

- 奇门：暗金 / 深墨；
- 八字：朱砂 / 土黄；
- 六爻：墨黑 / 竹青；
- 梅花：淡粉 / 青绿；
- 道家：青灰 / 月白；
- 易学基础：古铜 / 米白。

### BookCard

书籍卡要像“典籍”，不要像电商商品卡。

结构：

- 封面；
- 书名；
- 作者 / 朝代；
- 学科标签；
- 难度标签；
- 简介；
- 阅读人数；
- 查看详情。

视觉：

- 封面有轻微书脊阴影；
- 卡片边框淡金；
- hover 时封面轻微上移；
- 标题用宋体；
- 简介最多两行；
- 标签不要太多。

### ReaderSurface

用于阅读器正文纸面。

桌面端建议：

- 正文宽度 720px 到 780px；
- 背景 `paper-50`；
- 字号 18px；
- 行高 2；
- 段落间距 1.1em；
- 两侧留白充足。

移动端建议：

- 正文优先；
- 目录、术语、笔记放入底部抽屉；
- 不横向滚动；
- 顶部显示书名和章节。

### QimenShell

用于奇门工具区。

样式：

- 背景使用 `qimen-bg`；
- 正文使用 `qimen-text`；
- 面板使用 `qimen-panel`；
- 边框使用 `qimen-line`；
- 页面顶部仍保留全站导航，但工具内容区进入深色模式。

### QimenPanel

用于奇门输入区、结果区、盘理依据、报告预览等工具面板。

样式：

- 深墨面板；
- 暗金边框；
- 轻微内阴影；
- 标题用 `qimen-gold`；
- 辅助说明用 `qimen-muted`。

### NinePalaceChart

用于奇门九宫盘。

布局固定为：

```text
巽四 | 离九 | 坤二
震三 | 中五 | 兑七
艮八 | 坎一 | 乾六
```

每宫信息分三层：

1. 宫名 / 宫数；
2. 天盘干、地盘干、八门、九星、八神；
3. 空亡、入墓、门迫、击刑、吉格、凶格等状态标签。

状态标签要小而清楚，不使用大面积红绿。

### ReportCard

用于今日气机报告、一事择时报告、内容发布择时报告。

结构：

- 标题；
- 查询事项；
- 查询时间；
- 核心结论；
- 推荐时辰；
- 慎用时辰；
- 盘理依据；
- 行动建议；
- 风险提示；
- 免责声明；
- 品牌信息；
- 二维码占位。

报告文案禁止绝对化表达，统一使用“结构显示、倾向、较适合、建议、慎用、可作为参考”。

### 基础组件落地说明

当前阶段新增 `components/design/` 作为视觉系统的基础组件层。

已建立组件：

1. `PageShell`：统一页面背景、纹理、最大宽度和三套视觉空间。
2. `PageHero`：统一页面首屏结构，支持 `paper`、`qimen`、`minimal`。
3. `SectionTitle`：统一区块标题、小标签、说明文字和右侧操作。
4. `InkCard`：统一纸墨卡片、深色奇门卡片和后台卡片。
5. `SealBadge`：统一版权、会员、审核、草稿、私密、风险等标签。
6. `ScholarButton`：统一主按钮、次按钮、幽灵按钮和奇门按钮。
7. `TextureOverlay`：使用 CSS 渐变生成纸纹、九宫线和后台浅网格，不引入外部图片。
8. `EmptyState`：统一空数据、未登录、未上传、未生成报告等状态。

落地顺序：

1. 先用于 Header/Footer 的统一改造。
2. 再用于首页、藏书库、学科中心、奇门工具首页、我的书斋。
3. 后续再进入阅读器、后台、移动端专项和最终视觉验收。

迁移规则：

- 不一次性重写所有页面；
- 不改变业务数据和权限逻辑；
- 先替换页面骨架和基础组件，再逐步清理旧 class；
- 旧样式在未迁移页面中继续保留，避免视觉系统升级影响功能稳定性。

## 六、页面规范

### 首页

首页是平台门面，不堆功能，不做营销页。

结构建议：

1. 顶部导航：米白纸底、品牌红印、细分割线、当前导航朱砂短线；
2. 横卷视觉：使用宽幅古画或淡彩长卷作为第一视觉；
3. 书案入口：左侧说明“问古书斋”的定位，右侧展示六门学科卡片；
4. 精选藏书：展示 4 到 6 本，像书案上的典籍，不像电商商品；
5. 最近阅读：右侧或下方展示读书进度，形成个人书斋感；
6. 奇门工具：用深墨/暗金作为局部转场，不扩散到全站；
7. 平台能力：压成底部细栏或小卡，像“馆藏承诺”，不抢主视觉。

视觉重点：

- 米白纸色；
- 宋体标题；
- 朱砂印章；
- 淡金线条；
- 足够留白；
- 移动端不拥挤。

首页禁止：

- 首屏大面积黑绿遮罩；
- 竖排巨字压住横卷；
- 卡片过厚、阴影过重；
- 所有入口同等强调导致没有视觉层级；
- 把“奇门工具”做成全站主视觉。

### 藏书库

藏书库应像“数字藏书阁”，不是普通卡片列表。

结构建议：

- 顶部 PageHero；
- 搜索框；
- 书签式筛选；
- 书籍网格；
- 右侧最近阅读 / 阅读计划；
- 移动端筛选收起。

版权状态必须清楚：

- 公版；
- 自有；
- 授权；
- 摘录；
- 私密。

私密资料不显示“开始阅读”，只显示“仅后台可见”。

### 书籍详情

书籍详情页像“打开一本书之前的案头说明”。

结构建议：

- 左侧封面、收藏、开始阅读；
- 右侧书名、作者、朝代、学科、难度、版权状态；
- 简介；
- 适合人群；
- 学习重点；
- 目录；
- 相关术语；
- 相关课程；
- 相关工具；
- 读书笔记。

摘录资料只展示摘要和短摘；私密资料只展示基础档案。

### 阅读器

阅读器是长期体验核心。

桌面端：

- 左侧目录；
- 中间正文；
- 右侧术语 / 笔记；
- 正文最大宽度 760px；
- 原文宋体、行高 2；
- 白话解释使用清晰无衬线；
- 术语高亮用淡朱砂。

移动端：

- 正文优先；
- 目录、术语、笔记放到底部抽屉；
- 不横向滚动；
- 顶部显示书名和章节。

### 学科中心

学科中心是“入门地图”，不是普通分类页。

每个学科作为一扇门：

- 奇门：九宫、八门、九星、八神、三奇六仪；
- 八字：月令、气势、格局、调候、用神、大运流年；
- 六爻：世应、六亲、六神、用神、动变、应期；
- 梅花：体用、互变、外应；
- 道家：原文、白话、修身、文案素材；
- 易学基础：阴阳、五行、八卦、干支。

### 术语库

术语库要像“术数辞典”，不是标签列表。

结构建议：

- 搜索框；
- 学科筛选；
- 分类筛选；
- 术语卡片；
- 热门术语；
- 最近学习。

术语详情页包含：

- 原文解释；
- 白话解释；
- 专业解释；
- 应用场景；
- 相关条文；
- 相关案例；
- 相关工具。

### 课程

课程页更现代，但仍保持书斋气质。

结构建议：

- 课程 Hero；
- 课程筛选；
- 课程卡片；
- 模块目录；
- 课时列表；
- 相关书籍；
- 相关术语；
- 开始学习按钮。

课程学习页：

- 左侧课程目录；
- 中间课文；
- 右侧学习辅助；
- 底部上一课 / 下一课；
- 卡片不堆叠成后台仪表盘。

### 奇门工具

奇门工具页像进入“内室观盘”。

结构建议：

- QimenShell；
- 奇门问时 Hero；
- 工具卡片；
- 学习入口；
- 九宫盘；
- 盘理依据；
- 行动建议；
- 免责声明。

所有择时、报告和盘理文案必须稳妥：

- 结构显示；
- 倾向；
- 较适合；
- 建议；
- 慎用；
- 可作为参考。

禁止：

- 必准；
- 必成；
- 必发财；
- 改命；
- 保证有效；
- 绝对。

### 我的书斋

我的书斋是“个人学习案头”，不是普通用户中心。

结构建议：

- 顶部 Hero；
- 用户摘要卡；
- 最近阅读；
- 学习进度；
- 我的笔记；
- 我的报告；
- 功能入口：藏书、课程、排盘、报告、上传、设置。

视觉：

- 纸色卡片；
- 书笺感；
- 古铜 / 淡金进度条；
- SealBadge 显示会员、已读、草稿等状态。

### 后台管理

后台要清爽高效，不要过度古风。

结构建议：

- 左侧 Sidebar；
- 顶部 AdminHeader；
- 主内容区；
- 面包屑；
- 状态统计卡；
- 表格；
- 表单；
- 风险提示。

状态统一：

- 待处理：orange；
- 处理中：blue；
- 待审核：bronze；
- 已发布：green；
- 私密：muted；
- 风险：red。

后台必须明确提示：

> 未授权资料默认私密，不得公开全文展示；AI 生成内容必须审核后发布。

## 七、禁用风格

1. 不用廉价黑金。
2. 不用高饱和红黄。
3. 不堆太极、龙、符咒。
4. 不使用低清图片做大背景。
5. 不做强闪光动效。
6. 不写绝对化玄学承诺。
7. 不把全站做成预测软件。
8. 不把后台做成古风装饰页。
9. 不把阅读器做成普通 PDF 查看器。
10. 不使用“必准、改命、暴富、保证、绝对”等风险词。

所有页面最终都应回到同一个判断标准：

> 有书院气，有术数结构，有现代产品体验；专业、沉稳、可读、可学、可复盘。
