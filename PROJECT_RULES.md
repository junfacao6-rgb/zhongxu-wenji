# 项目开发规则（中虚书斋）

## 1. 项目定位

本项目是「传统术数知识库站」，不是普通 SaaS 官网。

目标：  
面向八字、六爻、奇门与传统文化学习者，提供：
- 古籍重排阅读
- 图解学习
- 工具速查
- 文章与课程内容整合

第一阶段先做结构完整、可本地运行的知识站，后续再接数据库/会员/支付。

## 2. 视觉风格与技术基调

- 视觉关键词：古籍、宣纸、克制、高级、东方、知识库、书斋
- 禁止：大面积紫色、霓虹、星座感、夸张符咒感
- 字体优先中文阅读体验：中文优先排版、适配移动端
- 色彩方向：米白/宣纸底 + 墨黑 + 朱砂红 + 淡金
- 组件风格：轻微阴影、细边框、圆角、节制留白

## 3. 研发原则（强制）

1. 所有模块先做静态可运行版本，不先引入后台复杂度。
2. 不一次性实现登录、支付、论坛、后台。
3. 组件必须拆分，结构清晰。
4. 数据结构可扩展，优先本地 JSON/TS 常量，保留后续数据库迁移路径。
5. 不删除现有功能，除非明确要求。
6. 每次修改后保留可运行状态。

## 4. 技术要求

- Next.js + TypeScript + Tailwind CSS
- 响应式优先（电脑/手机）
- 内容数据先用本地 JSON/MDX
- 避免过重依赖，优先稳定可部署方案
- 风格统一：以传统文化知识网站为主，不做“商业 SaaS 页面化”

## 5. 第一阶段范围（可交付）

必须交付页面：

- `/`
- `/library`
- `/learn`
- `/diagrams`
- `/tools`
- `/articles`
- `/articles/[slug]`
- `/library/[id]`

第一阶段工具：

1. 五行生克查询
2. 十神速查
3. 干支信息查询

说明：
- 前端本地计算
- 每个工具独立卡片与说明
- 选择天干/五行后即时显示结果
- 不只是表格展示，要有命理解释说明
- 代码结构为后续八字排盘、六爻起卦、奇门排盘留出扩展接口

## 6. 建议开发顺序

1. 首页 + 全局导航 + 视觉系统
2. 书架 + 文章 + 详情页
3. 图解模块
4. 工具模块
5. 移动端细化 + 部署

## 7. 与 AGENTS 的关系

本文件用于约束产品方向；  
`AGENTS.md` 保留工程指令（如运行方式、设计反馈记录约束）。

后续如有新目标或任务，我将优先按本文档拆分为：
